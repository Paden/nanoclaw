## Customizations to ADD

### IPC writer (host → session DB bridge)

**Intent:** Host-side slash hooks (`/wordle`, `/qotd`, `/chore`) need to wake the agent or post messages without going through `send_message`. v1 used JSON-file drops under `data/ipc/*/`; v2 has no watcher for that pattern.

**Files:** `src/ipc-writer.ts` (new file).

**How to apply:** Create `src/ipc-writer.ts` exporting `writeIpcMessage(groupFolder, msg)` and `writeIpcTask(groupFolder, task)`. The writer:

- Resolves `groupFolder` → `agent_group_id` → session via `getAgentGroupByFolder` (from `src/db/agent-groups.ts`) and `findSessionByAgentGroup` (from `src/db/sessions.ts`).
- For tasks: opens inbound.db via `openInboundDb` and calls `insertTask` (from `src/modules/scheduling/db.ts`) with `kind='task'`, `processAfter`, `content=JSON({prompt, script:null})`. Host-sweep wakes the container within 60s.
- For messages: calls `writeOutboundDirect` (from `src/session-manager.ts`) which inserts into outbound.db. Delivery loop picks it up; the channel adapter handles label/pin/upsert/unpin.
- Translates the JID scheme prefix (`dc:`, `tg:`, `sl:`) to `channelType` (`discord`, `telegram`, `slack`).
- For non-text action types (`unpin_message`, `delete_message`, etc.), writes `content.operation = type` so the adapter can dispatch; plain `message` type writes no operation tag.

Slash scripts (`scripts/qotd-slash.mjs`, `wordle-slash.mjs`, `chore-slash.mjs`) import this via `dist/ipc-writer.js`. Signatures match the legacy v1 stub, so the scripts don't need wiring changes.

### writeOutboundDirect Rw fix

**Intent:** `writeOutboundDirect` in `src/session-manager.ts` was opening outbound.db readonly and trying to INSERT — silently broken since v1→v2 migration. Same path is also needed in `drainSession` (delivery.ts) for hot-journal recovery from killed containers.

**Files:** `src/session-manager.ts`, `src/delivery.ts`.

**How to apply:**

- In `session-manager.ts:writeOutboundDirect`, change the `openOutboundDb` call to `openOutboundDbRw`.
- In `delivery.ts:drainSession`, change `openOutboundDb` import + call to `openOutboundDbRw`. With `journal_mode=DELETE` and `busy_timeout=5000` (set inside `openOutboundDbRw`), the host RW handle is safe to hold concurrently with a writing container.
- Update the doc comment on `openOutboundDbRw` to reflect the actual concurrency story (it's safer than the original "only when no container is running" implies).

### Container idle-recycle ceiling tuning

**Intent:** 30-min hard idle ceiling caused frequent cold-start churn on groups that wake hourly. Bumped default to 90 min, with env override. Also demoted the kill log from `warn` to `info` since it's routine recycling.

**Files:** `src/host-sweep.ts`.

**How to apply:**

- `ABSOLUTE_CEILING_MS = parseInt(process.env.CONTAINER_TIMEOUT || '5400000', 10);` (was hardcoded `30 * 60 * 1000`).
- Change `log.warn('Killing container past absolute ceiling', ...)` → `log.info('Recycling idle container past absolute ceiling', ...)` with a clarifying comment.

### discord adapter: `unpin_message` action handler

**Intent:** Retiring pinned cards required an unpin path. The adapter previously handled label/pin/upsert during normal `deliver()` but had no way to retire a pin. Added an `unpin_message` action that reads `data/sessions/<groupFolder>/message_labels.json`, fetches the stored message ID, unpins it via Discord.js `msg.unpin()`, and clears the label entry. Handles both `{id, date}` object shape and bare-string label values.

**Files:** `src/channels/discord.ts`.

**How to apply:** Add a new branch at the top of `deliver()` (before the reaction-operation branch) that fires when `content.operation === 'unpin_message'`. Read labels JSON, look up id, unpin and clean up.

## v2 IPC pattern (architecture context, not a single change)

**Why this matters:** The original guide was written when host-side slash hooks dropped JSON files into `data/ipc/<group>/{tasks,messages}/` and a watcher picked them up. v2's architecture (per CLAUDE.md: "There is no IPC, no file watcher, no stdin piping between host and container. The two session DBs are the sole IO surface.") killed the watcher. Result: 123 stale JSON files piled up over weeks while reveals silently failed.

**Resolution applied:** rewrote `src/ipc-writer.ts` (see "IPC writer (host → session DB bridge)" above). All three slash hooks now route through that. When applying the migration, ensure the new IPC writer goes in BEFORE attempting to use any slash-command hook.
